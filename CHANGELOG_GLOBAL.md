## 2026-04-07: Active-path cleanup and scene hardening

### Runtime safety
- `TicketManager` переписан под текущий active scope.
- Архивные билеты больше не участвуют в рабочей логике:
  - не выдаются через active API
  - не спавнятся как physical tickets
  - если старый билет всё же попадёт в руку, он безопасно сбрасывается
- Рабочими билетами остаются только `Inspection` и `MedicalRation`.

### Card logic
- `CreateRandomMainCard(bool allowThreat)` теперь реально учитывает `allowThreat`.
- Раундовые стартовые руки остаются:
  - Раунд 1: `4`
  - Раунд 2: `5`
  - Раунд 3: `6`
- Колода по-прежнему создаётся в памяти, раздаёт случайно, а при полном опустошении пересоздаётся для того же раунда.

### Scene wiring
- `BCHandAnimator` получил прямые holder-ссылки в `MainSceneBuilder`.
- `Assets/Scenes/mainScene.unity` обновлена так, чтобы hand holders были прописаны напрямую и не зависели только от поздней runtime-инициализации.

## 2026-04-07: Playable runtime stabilization and round-hand update

### Card loop
- Стартовая рука основных карт теперь зависит от раунда:
  - Раунд 1: `4`
  - Раунд 2: `5`
  - Раунд 3: `6`
- Основная колода создаётся в памяти для текущего раунда и затем из неё случайно раздаются карты игроку и врагу.
- Если колода текущего раунда опустела, создаётся новая колода того же раунда без сброса текущего состояния матча.
- Добор по-прежнему происходит только стороне, у которой основная рука опустела полностью, и даёт `3` карты.

### Gameplay scope
- Карты врага скрыты и должны отображаться как `?`.
- Рабочий набор спецкарт сейчас ограничен `Cancel` и `Duplicate`.
- Рабочая генерация билетов сейчас ограничена `Inspection` и `MedicalRation`.
- Выигрыш O2 за выстрел использует текущее правило:
  - попадание с уроном по противнику: `+30s`
  - выстрел без урона: `+10s`

### Scene and tooling
- Добавлен активный генератор сцены:
  - `Breath Casino/Main Scene/Rebuild mainScene`
- Сгенерирована `Assets/Scenes/mainScene.unity` вокруг `mainScene.fbx`.
- Устаревшие blockout-editor утилиты переведены в архивный режим и больше не должны участвовать в рабочем пайплайне.

### Architecture
- Первый безопасный шаг разгрузки `GameManager` сохранён как внутренняя HUD-структура в `GameManager.cs`, чтобы не ломать `dotnet build` до полной регенерации проектных файлов Unity.
- `BCAudioManager` расширен до более гибкой cue-based схемы с сохранением совместимости по публичным вызовам.

### Docs
- Обновлена активная истина в:
  - `Assets/MECHANICS.md`
  - `Assets/PLANS.md`
  - `CLAUDE.md`
