  2. Дебаг-худ — как это работает и как убрать

  Дебаг-текст рисуется через обычный Unity Text компонент на Canvas. Цепочка
  зависимостей:

  SceneBootstrap
    └── public Text statusText       ← Unity UI Text (старый, не TMP)
          │
          └── SetStatus(string)
                │
                └── GameManager.RefreshHud()  ← вызывается ~30+ раз по всему
  коду
                      └── builder собирает: Phase, HP, O2, Chamber, tickets,
  last event...
                            └── _scene.SetStatus(builder.ToString())

  statusText — это поле в SceneBootstrap (строка 20), которое ты должен
  привязать в Inspector к какому-то Text объекту в сцене. Если этот Text объект
  виден в Game View — ты видишь хад.

  Как убрать/спрятать:

  Вариант A — самый простой (в Unity Editor):
  Найди в иерархии сцены Canvas → внутри него Text объект привязанный к полю
  Status Text на SceneBootstrap. Отключи (SetActive false) или удали этот Text
  GameObject. Поле statusText в SceneBootstrap станет null — SetStatus()
  проверяет if (statusText != null), так что краш не будет.

  Вариант B — временно скрыть:
  На SceneBootstrap в Inspector просто очисти поле Status Text (перетащи ни на
  что). Тот же эффект.

  Вариант C — если хочешь оставить для дебага но не показывать игроку:
  Переведи Text на отдельный Layer, который не рендерится основной камерой (или
  поставь alpha = 0 на Canvas Group).
